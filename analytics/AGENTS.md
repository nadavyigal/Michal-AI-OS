# הוראות לסוכן — אוטומציית Wix Analytics

אתה עובד בפרויקט אוטומציית ביצועי האתר של מיכל.

## תכנון לפני ביצוע — חובה

לפני כל פעולה שמשנה קבצים, מריצה סקריפט, מושכת נתונים או כותבת תוצר:
1. הצג תוכנית קצרה: מה תעשה, אילו קבצים/endpoints ייגעו, ומה התוצר.
2. חכה לאישור ("אשר" / "המשך") לפני ביצוע.
3. אם משהו לא ברור — שאל עד 3 שאלות לפני שתתחיל.
משימות קריאה בלבד (סיכום, בדיקה) אפשר לבצע ישר, אבל תמיד לומר מה נמצא לפני שממשיכים לשינויים.

## Wix — חקירה עצמאית (המשימה המרכזית)

מקור הנתונים ל-Wix עדיין לא סגור. תפקידך לחקור ולפתור אותו בעצמך:
1. חקור את Wix Analytics Data API בתיעוד הרשמי (dev.wix.com): אילו endpoints קיימים, אילו measurement types רלוונטיים (visits, unique visitors, page views, leads/contacts), ואיך מבצעים אימות (API key או OAuth + Site ID + הרשאת Site Analytics Read).
2. הצג תוכנית לפני קוד: מאיזה endpoint תמשוך, אילו שדות, ולאיזה טווח תאריכים.
3. השלם את scripts/fetch_wix_analytics_api.py (כרגע שלד עם TODO) לפי מה שמצאת.
4. אם אין עדיין credentials בקובץ .env — תכנן את מסלול ה-API אבל עבוד בינתיים מ-CSV בתיקיית data, וציין במדויק אילו credentials מיכל צריכה להשיג ומאיפה.
אל תמציא מספרים. אם אין נתונים אמיתיים — אמור זאת במפורש.

## תפקיד

לבנות ולתחזק תהליך שמביא נתוני Wix Analytics באופן אוטומטי, מכניס אותם לטבלה, ומפיק סיכום יומי/שבועי בעברית.

## שפה

תמיד לעבוד בעברית.

## מותר

- לעבוד עם Wix Analytics API
- לעבוד עם CSV של דוחות Wix
- לעבוד עם Google Sheets של דוח ביצועים
- ליצור סיכומים עסקיים בעברית

## אסור

- לערבב נתוני מתאמנים/גבייה
- להכניס שמות לקוחות או טלפונים
- להשתמש בקבצי `02-billing-local`

## מקורות נתונים אפשריים

1. עדיף: Wix Analytics Data API
2. גיבוי: מיילים/CSV אוטומטיים מ-Wix Reports
3. גיבוי נוסף: CSV ידני בתיקיית `data`

## משימת התחלה מומלצת

1. קרא את `README.md`.
2. בדוק אם קיים קובץ `.env`.
3. אם אין API token, עבוד דרך CSV בתיקיית `data`.
4. צור `outputs/summary_latest.md` בעברית.
