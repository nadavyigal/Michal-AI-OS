# תוכנית הקמה מורחבת למערכת ה-AI של מיכל

תאריך: 7 ביולי 2026

המטרה: להפוך את העבודה של מיכל למערכת פשוטה וברורה בעברית, עם הפרדה בין תוכן, חשיבה אסטרטגית, ביקורת, גבייה מקומית ודוחות אתר.

## מבנה המערכת

1. Google Drive  
   מאגר חומרים לא רגישים: קול של מיכל, מסמכי תוכן, עמודי נחיתה, אסטרטגיית תוכן, הוראות עבודה.

2. NotebookLM  
   מוח ידע שמבוסס על מקורות. לא מבצע אוטומציות ולא נוגע בקבצי גבייה רגישים.

3. Gemini Gems  
   כלי ביצוע מהירים בעברית: כתיבת פוסטים, ניוזלטרים, מחזור תוכן, טיוטות גבייה אנונימיות.

4. Antigravity  
   שכבת עבודה אקטיבית: קבצים מקומיים, אקסלים, סקריפטים, אוטומציות Wix, בדיקות ותוצרים.

5. Obsidian, בהמשך  
   זיכרון עבודה קבוע: החלטות, סיכומי יום, תובנות, תהליכים, Wiki פנימי.

## פרויקטים ב-Antigravity

כבר הוכנו שלושה פרויקטים:

1. `01-content-os`  
   תוכן, עמודי נחיתה, ניוזלטרים, רעיונות לפוסטים ומחזור תוכן.

2. `02-billing-local`  
   גבייה ואקסלים מקומיים בלבד. לא מעלים לכאן חומרים מ-Drive ולא מוציאים מכאן פרטי לקוחות.

3. `03-wix-analytics`  
   אוטומציה לדוחות Wix, קריאת נתונים, סיכום יומי והזנה עתידית ל-Google Sheet.

## מחברות NotebookLM מומלצות

להקים ארבע מחברות:

1. `מיכל - Content OS`  
   מחברת התוכן הראשית.

2. `מיכל - STORM`  
   מחברת מחקר, סיעור מוחות, זיהוי הזדמנויות ובניית כיוונים.

3. `מיכל - ADVISOR`  
   מחברת ייעוץ עסקי ואסטרטגי לקבלת החלטות.

4. `מיכל - RED TEAM`  
   מחברת ביקורת, בדיקת סיכונים, איכות, פרטיות וחורים במסרים.

לא להכניס לאף מחברת NotebookLM קבצי גבייה, שמות מתאמנים, טלפונים, מיילים, תיעוד טיפולי או אקסלים רגישים.

## פרומפטים להרצה

הפרומפטים המדויקים להדבקה בכל פרויקט Antigravity נמצאים בקובץ נלווה: `SETUP-PROMPTS-ANTIGRAVITY.md`. גם בכל פרויקט יש תיקיית `prompts` עם הפרומפטים שלו (`משימות-תוכן.md`, `משימות-אתר.md`, `משימות-גבייה.md`).

## "סקילז" ותוספים ב-Antigravity

ב-Antigravity אין חנות תוספים כמו Obsidian. שלושת מנגנוני ההרחבה הם:

1. **AGENTS.md בכל פרויקט** — זה ה"סקיל" האמיתי. מגדיר תפקיד, שפה, מותר/אסור, ותוצרים. כבר כתוב בשלושת הפרויקטים.
2. **קבצי prompts** — פרומפטים חוזרים שנשמרים בפרויקט ומודבקים לפי צורך. כבר קיימים.
3. **שרתי MCP / כלים חיצוניים** — להוסיף רק כשצריך יכולת שאין מקומית.

### MCP שכדאי לשקול (בהדרגה, לא הכל ביחד)

- **Filesystem + הרצת Python** — מובנה. מספיק לתוכן, גבייה, וקריאת CSV. אין מה להוסיף עכשיו.
- **Google Sheets MCP** — רק בשלב האוטומציה, כדי לדחוף את הדוח היומי לגיליון. עד אז לא צריך.
- **Playwright / browser MCP** — רק כגיבוי אם Wix API לא מספק את הנתונים. לא להתקין מראש.

כלל: לא מוסיפים MCP "ליתר ביטחון". כל שרת MCP הוא קוד צד שלישי עם גישה לפרויקט — מוסיפים אחד בכל פעם, רק כשיש צורך אמיתי, בדיוק כמו תוספי קהילה ב-Obsidian.

## תוספים וכלים מומלצים

### חובה בשלב הנוכחי

- Google Drive: אחסון מסמכים לא רגישים.
- NotebookLM: עבודה מבוססת מקורות.
- Gemini Gems: תבניות עבודה חוזרות בעברית.
- Antigravity: עבודה מקומית עם קבצים, אקסלים ואוטומציות.
- Python בפרויקטים המקומיים: קריאת CSV, XLSX, ODS והפקת דוחות.

### מומלץ להוסיף לשלב האוטומציה

- Google Sheets API: כדי לעדכן דוח ביצועים יומי בלי עבודה ידנית.
- Wix Analytics API: כדי למשוך נתוני אתר ישירות מ-Wix.
- Gmail או מייל אוטומטי: רק אם מחליטים לעבוד דרך דוחות Wix שנשלחים למייל.
- Browser automation / Playwright: רק כפתרון גיבוי אם Wix API לא מספיק.
- Cron או launchd במחשב של מיכל: להרצת דוח יומי בשעה קבועה.

### מומלץ לשלב Obsidian בהמשך

להתחיל עם תוספי Core בלבד:

- Daily notes
- Templates
- Backlinks
- Graph view

אם צריך תוספי קהילה, להוסיף בהדרגה בלבד:

- Dataview: שאילתות על הערות ומשימות.
- Tasks: ניהול משימות מתוך הערות.
- Templater: תבניות מתקדמות.
- Periodic Notes / Calendar: יומן יומי, שבועי וחודשי.
- Omnisearch: חיפוש טוב יותר.
- Obsidian Git: גיבוי גרסאות, רק אם יודעים להגדיר נכון.

הערת אבטחה: תוספי קהילה ב-Obsidian הם קוד צד שלישי. מתקינים רק תוספים מוכרים, אחד בכל פעם, ורק אם יש צורך אמיתי.

## רוטינות יומיות מומלצות

### כל בוקר - דוח אתר

מטרה: לדעת מה קרה באתר אתמול בלי שמיכל תיכנס ידנית ל-Wix.

תוצר יומי:

- כניסות לאתר
- מבקרים ייחודיים
- לידים / הרשמות / לחיצות WhatsApp
- שיעור המרה
- עמודים מובילים
- מקורות תנועה
- חריגות או שינוי משמעותי מאתמול
- המלצה אחת לפעולה

בשלב ראשון מפעילים מתוך Antigravity. בשלב הבא מחברים ל-Wix API ול-Google Sheet.

### כל יום - תוכן

מטרה: לא להמציא מחדש את התוכן כל יום.

תהליך:

1. רעיון או נושא מהיום.
2. Content OS או STORM מחדד זווית.
3. Gem תוכן כותב טיוטה.
4. RED TEAM בודק אם המסר חד, אמין ולא גנרי.
5. התוצר נשמר ב-Obsidian או בתיקיית outputs של Antigravity.

### כל יום - גבייה

מטרה: לראות מצב גבייה בלי לחשוף פרטים רגישים.

תהליך:

1. קבצי האקסל נשמרים רק ב-`02-billing-local/data`.
2. הסוכן המקומי קורא את הקבצים.
3. מוציא רשימת חריגים אנונימית.
4. רק אחרי אישור אנושי מכינים הודעות לפי כינויים.

### פעם בשבוע - ישיבת OS

משך מומלץ: 30 דקות.

סדר:

1. ADVISOR: מה שלושת הדברים הכי חשובים השבוע?
2. Wix Analytics: מה עבד ומה לא?
3. Billing Local: האם יש חריגות גבייה?
4. Content OS: מה לפרסם השבוע?
5. RED TEAM: מה עלול להיות בעייתי?
6. Obsidian: לשמור החלטות ותובנות.

## שלבי ביצוע

### שלב 1 - היום

1. לפתוח את שלושת פרויקטי Antigravity במחשב של מיכל.
2. להוריד את קובץ הזיפ מה-Drive.
3. לשים את האקסלים הרגישים רק בפרויקט `02-billing-local/data`.
4. להקים שלוש מחברות NotebookLM חדשות: STORM, ADVISOR, RED TEAM.
5. להדביק לכל מחברת את ההוראות המתאימות מהתיקייה `notebooklm-advisors`.

### שלב 2 - השבוע

1. לחבר Wix API או להגדיר דוחות CSV אוטומטיים מ-Wix.
2. להפוך את דוח האתר ליומי, קבוע ובעברית.
3. לבדוק שהסוכן המקומי קורא את האקסלים בלי לחשוף שמות.
4. להריץ שבוע ניסיון עם רוטינת בוקר ורוטינת תוכן.

### שלב 3 - אחרי שהמערכת עובדת

1. להקים Obsidian Vault בשם `Michal OS Vault`.
2. ליצור Wiki פנימי.
3. להוסיף תבניות Daily / Weekly / Decision / Content Brief.
4. לסנכרן רק חומרים לא רגישים.
5. להשאיר גבייה ופרטי לקוחות מחוץ ל-Vault אם אין צורך אמיתי.
