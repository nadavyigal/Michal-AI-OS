# Social Media Manager — Michal

This is the **strategy layer** that sits on top of Track C's Gems. Track C built the
*assistants* (Content/Voice, Newsletter, Repurpose, Billing); this is the actual *strategy*
those assistants execute against — who she's talking to, what she stands for, what she posts
about, when, with what hooks, how she engages, and how she'll measure it.

Built by running the 7-step social-media-manager workflow Nadav specified, grounded entirely
in what Track C already mined (the Voice doc, her own real pain-point copy, her real content-
mix percentages, the dual-practice finding) — not generic framework filler.

## How this connects to Track C

```
social-media-manager/  (this folder — WHAT to post, WHEN, WHY)
        │
        ▼
gems/content-voice-gem.md  (Track C — HOW to write it, in her actual voice)
        │
        ▼
published post
```

The calendar (file 04) and hook bank (file 05) give the **concept**. Running that concept
through the live Content/Voice Gem produces the **final, ready-to-publish copy**. This folder
doesn't replace the Gem — it feeds it.

## Honest caveats (read before presenting tonight)

- **Step 7 cannot run yet.** We have no real Instagram/Facebook metrics. File 07 is the
  framework she runs once she shares an Insights export — it's a template, not a finished
  analysis. Don't present it as more than that.
- **Visual identity (file 02) is provisional**, pending tonight's confirmation of which palette
  is current (per the Voice doc's open question #3).
- **The audience split (file 01) surfaces a real strategic decision, not just analysis**:
  running couples-coaching and management-coaching content on one feed risks diluting both.
  This is a genuine open question for her, not a settled recommendation — raise it as a
  question tonight, not a verdict.
- **Privacy rule applies here too.** File 06's "share anonymized client insights" tactic must
  stay at the pattern level — no identifying detail, ever, even an indirectly identifiable
  combination (age + role + quote can de-anonymize in a small community). Same red line as
  every Gem in Track C.
